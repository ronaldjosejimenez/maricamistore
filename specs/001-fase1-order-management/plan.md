# Implementation Plan: Fase 1 — Sales & Order Management System

**Branch**: `001-fase1-order-management` | **Date**: 2026-06-03 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/001-fase1-order-management/spec.md`

## Summary

Build a complete multi-tenant sales and order management system for MariCamiStore on top of an existing ASP.NET Core 10 Razor Pages project. The domain models and EF Core infrastructure already exist; work focuses on adding multi-tenancy infrastructure, completing CRUD views for all catalogs, implementing the order lifecycle with business logic, automating transaction ledger entries, and delivering the payment registry and accounts receivable report.

Implementation follows an incremental layer approach: infrastructure → catalogs → order management + business logic → reports.

---

## Technical Context

**Language/Version**: C# 13 / .NET 10

**Primary Dependencies**: ASP.NET Core 10 Razor Pages, Entity Framework Core 10, SQL Server, AdminLTE 3.0.5, jsGrid, Bootstrap 4, jQuery

**Storage**: SQL Server via EF Core Code-First (existing migrations present)

**Testing**: Out of scope for Phase 1

**Target Platform**: Windows (development) / Linux container (future Azure deployment)

**Project Type**: Web application (Razor Pages + AJAX)

**Performance Goals**: CRUD operations < 2s; real-time recalculation < 500ms (client-side JS only)

**Constraints**: UI in Spanish; all code in English; no authentication in Phase 1; multi-tenant data isolation mandatory

**Scale/Scope**: Single-user per session; small business scale (hundreds of orders, tens of customers)

## Implementation Layers

### Layer 1 — Multi-Tenant Infrastructure

**Goal**: Every request knows the active `OrganizationId`; all queries filter automatically.

1. Add session services to `Program.cs` (`AddSession`, `UseSession`)
2. Create `ICurrentOrganizationService` + `CurrentOrganizationService`:
   - Reads `OrganizationId` (Guid) from `IHttpContextAccessor` session key `"ActiveOrganizationId"`
   - Returns `Guid.Empty` when no org selected
3. Add Global Query Filters to `MariCamiStoreContext`:
   - `Order`, `OrderItem`, `Transaction` → filter by `OrganizationId == _currentOrg.OrganizationId`
   - `Customer`, `Currency`, `ProductType`, `Supplier`, `Configuration`, `Organization` → no filter (global catalogs or tenant-defining entities)
4. Modify `_Layout.cshtml`:
   - Navbar dropdown listing all Organizations; selected org highlighted
   - On change: AJAX POST to `/api/session/organization` → sets session → page reload
   - Global jQuery AJAX spinner: `$(document).ajaxStart` / `ajaxStop` show/hide overlay div
5. Add `OrderStatusHistory` entity, EF config, and migration

### Layer 2 — Catalog CRUDs

**jsGrid handler pattern** (same for all catalogs):

| Handler | HTTP | jsGrid event | Returns |
|---------|------|--------------|---------|
| `OnGetAsync` | GET | page load | Page() |
| `OnGetLoadAsync` | GET | loadData | `JsonResult(list)` |
| `OnPostInsertAsync` | POST | insertItem | `JsonResult(newEntity)` |
| `OnPostUpdateAsync` | POST | updateItem | `JsonResult(updatedEntity)` |
| `OnPostDeleteAsync` | POST | deleteItem | `JsonResult({ success=true })` |

**Catalogs to implement**:
- `Configurations/Index` — single-record (upsert pattern; no delete)
- `Currencies/Index` — standard CRUD
- `ProductTypes/Index` — standard CRUD; includes EstimateShipping, ServiceFeeInLocal, CurrencyId
- `Suppliers/Index` — standard CRUD
- `Customers/Index` — standard CRUD (no org filter applied)

---

### Layer 3 — Order Management & Business Logic

#### Order Dashboard (`Orders/Index`)
- jsGrid: loads orders with default filter `Status IN ('Pending','Active')`; status dropdown to show all
- Create order: modal form → `OnPostCreateAsync` → returns new order
- Status transition: per-row button opens modal with TransitionDate + Notes + Justification (required only for Voided) → `OnPostTransitionAsync`
- `OnPostTransitionAsync` validates state machine, saves `OrderStatusHistory`, runs transaction automation atomically

#### Order Items Editor (`Orders/Items?orderId={id}`)
- Header: read-only order summary (name, supplier, status, totals)
- jsGrid: items CRUD — enabled only if `Order.Status == Pending`
- On `ProductTypeId` change (JS): `GET /Orders/Items?handler=ProductType&id={id}` → auto-fills EstimateShipping, ServiceFeeInLocal
- Real-time JS recalculation on every numeric field change:
  - `ListPriceTax = ListPrice × (TaxPercentage / 100)`
  - `AgreedPriceInLocal = (ListPrice + ListPriceTax) × ExchangeRate + ServiceFeeInLocal`
  - Order header totals recalculate from all rows in the grid
- On item save: server recomputes `ShippingAmountToCR = Sum(EstimateShipping)` and persists updated Order header

#### Delete Rules (enforced in `OrderService`)
- `DeleteOrderAsync`: only allowed when `order.Status == Pending`. Cascades deletion of all items. Returns `(bool Success, string? Error)`.
- `DeleteOrderItemAsync`: only allowed when parent `order.Status == Pending`. Returns `(bool Success, string? Error)`. The dashboard shows a delete button only for Pending orders; the items editor blocks the action server-side with a user-facing error.

#### Transaction Automation (inside `OrderService`)
- `TransitionOrderAsync(orderId, toStatus, date, notes, justification)`:
  - Validates: allowed transition, non-empty justification for Voided, ≥1 item for Active
  - Creates `OrderStatusHistory` record
  - If `toStatus == Active`: foreach item → insert `Transaction { Type=Charge, Amount=item.AgreedPriceInLocal, CustomerId=item.CustomerId, SourceId=item.Id }`
  - If `toStatus == Voided` (from non-Pending): foreach item → insert `Transaction { Type=Void, Amount=item.AgreedPriceInLocal, CustomerId=item.CustomerId, SourceId=item.Id }`
  - All inserts in one `SaveChangesAsync()` call

---

### Layer 4 — Reports

#### Payment Registry (`Payments/Index`)
- Customer dropdown; on select: AJAX `OnGetBalanceAsync(customerId)` → returns `{ globalBalance, orgBalance }`
- Amount input; on submit: `OnPostRegisterPaymentAsync` → creates `Payment` Transaction → returns updated balances

#### Saldos Report (`Reports/Saldos`)
- `OnGetAsync`: query groups Transactions by CustomerId, calculates `Sum(Charge) - Sum(Payment) - Sum(Void)`, filters `balance > 0`, joins Customer name
- Renders as static HTML table (no grid, no AJAX needed)
