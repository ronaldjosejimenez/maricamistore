# Implementation Plan: Mejoras Fase 1

**Branch**: `001-fase1-order-management` | **Date**: 2026-06-03 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/002-mejoras-fase1/spec.md`

---

## Summary

Incrementally improve the MariCamiStore system built in Fase 1 with six targeted enhancements:

1. Auto-load the active organization from `appsettings.json` on first request (no manual selection required)
2. Add an organization guard to all dependent screens via a shared `OrganizationPageModel` base class
3. Fix currency display in Configurations and ProductTypes grids (show abbreviation instead of raw GUID)
4. Add full CRUD for Organizations via a new `IOrganizationService` with referential-integrity enforcement
5. Enforce one-configuration-per-organization in the UI (disable insert mode when config exists)
6. Improve the Order creation form: Supplier field first, auto-suggest name, auto-load ExchangeRate and TaxPercentage from org config, and confirm items are blocked when order is not Pending

No new database migrations are required.

## Implementation Layers

### Layer 1 — Default Organization from Appsettings

**Files**: `Settings.cs`, `appsettings.json`, `CurrentOrganizationService.cs`

1. Add `Guid? DefaultOrganizationId` to `BaseSettings`.
2. Add `"DefaultOrganizationId": null` to `appsettings.json` (and Development/Production variants if needed).
3. Inject `IOptions<BaseSettings>` into `CurrentOrganizationService`.
4. In `CurrentOrganizationService.OrganizationId` getter: if session has a value, return it; if not AND `DefaultOrganizationId` is set, write it to session and return it.
   - Note: writing to session requires `IHttpContextAccessor.HttpContext.Session.SetString(...)`. This is already available in the constructor chain.

---

### Layer 2 — Organization Guard (OrganizationPageModel)

**Files**: `Pages/Shared/OrganizationPageModel.cs`, affected page models

1. Create `OrganizationPageModel : PageModel` with:
   - Constructor injects `ICurrentOrganizationService`
   - Virtual `OnGet()` that checks `CurrentOrg.OrganizationId == Guid.Empty` → `RedirectToPage("/Organizations/Index", new { error = "..." })`
   - Pages override `OnGet()` and call `base.OnGet()` first (or return its result)
2. Change `Configurations/Index.cshtml.cs`, `Payments/Index.cshtml.cs`, `Reports/Saldos.cshtml.cs` to inherit `OrganizationPageModel`.
3. Migrate `Orders/Index.cshtml.cs` and `Orders/Items.cshtml.cs` to inherit `OrganizationPageModel` (removes duplicated guard code).

---

### Layer 3 — Organizations CRUD

**Files**: `IOrganizationService.cs` (new), `OrganizationService.cs` (new), `Organizations/Index.cshtml.cs` (modify), `ApplicationExtensions.cs` (register)

**`IOrganizationService` interface**:
```csharp
Task<List<Organization>> GetOrganizationsAsync();
Task<Organization> CreateOrganizationAsync(Organization org);
Task<Organization> UpdateOrganizationAsync(Organization org);
Task<(bool Success, string? Error)> DeleteOrganizationAsync(Guid id);
```

**`DeleteOrganizationAsync` logic**:
- Check if any `Order.OrganizationId == id` → return `(false, "No se puede eliminar...")`
- Check if any `Configuration.OrganizationId == id` → return `(false, "No se puede eliminar...")`
- Otherwise, delete and return `(true, null)`

**`Organizations/Index.cshtml.cs` refactor**:
- Remove `MariCamiStoreContext` injection; inject `IOrganizationService`
- Keep `OnGetLoadAsync` and `OnPostSetActiveAsync`
- Add `OnPostInsertAsync([FromBody] Organization item)`, `OnPostUpdateAsync([FromBody] Organization item)`, `OnPostDeleteAsync([FromBody] DeleteRequest request)`

**`wwwroot/js/pages/organizations/index.js`** — convert to full jsGrid CRUD (matching pattern of other catalog pages), keeping the SetActive button per row.

---

### Layer 4 — Currency Display Fix

**Files**: `wwwroot/js/pages/configurations/index.js`, `wwwroot/js/pages/product-types/index.js`

For both files:
1. On `$(function() {...})` init, first load currencies from `/Currencies?handler=Load`.
2. Map response to `currencyItems = data.map(c => ({ id: c.id, text: c.abbreviation }))`.
3. Call `initGrid()` only after currencies are loaded.
4. Replace the text column for currency ID with jsGrid `select` field:
   ```js
   { name: 'localCurrencyId', title: 'Moneda Local', type: 'select', items: currencyItems, valueField: 'id', textField: 'text', width: 120 }
   ```

**`configurations/index.js` additional change — one-per-org enforcement**:
- After `loadData` completes, check result length:
  - 0 → `$('#jsGrid').jsGrid('option', 'inserting', true); $('#config-banner').hide()`
  - ≥1 → `$('#jsGrid').jsGrid('option', 'inserting', false); $('#config-banner').show()`
- Add a `<div id="config-banner" class="alert alert-info" style="display:none">Esta organización ya tiene una configuración. Solo puede editarla.</div>` in the Configurations/Index.cshtml above the grid.

### Layer 5 — Order Form Improvements

**Files**: `Orders/Index.cshtml.cs` (add handler), `Orders/Index.cshtml` (reorder fields), `wwwroot/js/pages/orders/index.js`

**Backend — new handler in `Orders/Index.cshtml.cs`**:
```csharp
public async Task<JsonResult> OnGetConfigurationAsync()
{
    var config = await catalogService.GetConfigurationAsync();
    return new JsonResult(new { config?.ExchangeRate, config?.TaxPercentage });
}
```
Inject `ICatalogService` into `Orders/Index` constructor.

**HTML (`Orders/Index.cshtml`) — modal field order**:
Move `#order-supplier` select group to be the first field inside the modal form body.

**JS changes in `orders/index.js`**:
1. `openNewOrder()`: after clearing fields, call `$.get('?handler=Configuration', function(cfg) { if (cfg) { $('#order-exchange-rate').val(cfg.exchangeRate || ''); $('#order-tax').val(cfg.taxPercentage || ''); } })`.
2. Add flag `var userEditedName = false;` at file scope, reset to `false` in `openNewOrder()`.
3. Add `$('#order-name').on('input', function() { userEditedName = true; })`.
4. Add `$('#order-supplier').on('change', function() { if ($('#order-id').val() || userEditedName) return; var supplierName = $(this).find('option:selected').text(); if (!supplierName) return; var d = new Date(); var dd = String(d.getDate()).padStart(2,'0'); var mm = String(d.getMonth()+1).padStart(2,'0'); var yyyy = d.getFullYear(); $('#order-name').val(supplierName + '-' + dd + '-' + mm + '-' + yyyy); })`.
