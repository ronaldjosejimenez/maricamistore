## 5. Validation Rules

| Field | Rule |
|---|---|
| `Customer.IsGeneric` | Non-nullable. Default `false`. Only "Sin Cliente" has `true`. No UI exposes this field directly. |
| `SaldoReportRow.IsGeneric` | Populated from `Customer.IsGeneric` at query time. Drives "(Especulativo)" label in the saldos UI. |

## 6. No-Change Confirmation

The following fields/tables must NOT be modified as part of this feature:

- `OrderItem.CustomerId` — already `Guid` (non-nullable), accepts "Sin Cliente" ID without schema change
- `Order.TotalToPayToSupplier`, `TotalOfTheOrder`, `TotalWithoutTaxes`, `TaxesAmount`, `ShippingAmountToCR` — invariant during reassignment (FR-012)
- `Transaction` table — no schema changes; new rows created via `ReasignarItemAsync`
