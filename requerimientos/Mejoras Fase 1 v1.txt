### 1. General UI & Structural Changes
- **Item Form:** Since there are too many fields, replace any inline item addition with a dedicated Form/Modal for adding and editing order items.
- **Order Editing Mode:** Remove the separate edit dialog for the Order. Editing must happen directly in-place within the order summary screen.
- **State-based Editability:** The order screen should remain in "edit mode" constantly as long as the order status is `Pending`. 
- **UI Warning:** Display a prominent label/legend saying: "Presionar guardar para recalcular si hay algo pendiente de guardar" (Press save to recalculate if there are pending unsaved changes).

### 2. Order Item Fields & Logic Updates (Grid / Form / Query)
Apply the following rules to the Order Items:

* **CustomerId:** Keep as-is, but implement a searchable Combobox/Dropdown (search-as-you-type). *Must be included in the Query view.*
* **ProductDescription:** Keep as-is. *Must be included in the Query view.*
* **ProductLink:** Add a new **optional (nullable)** text field.
* **ProductSourceCode:** Add a new **optional (nullable)** text field. *Must be included in the Query view.*
* **ProductImage:** Add an image upload field. Convert the uploaded image to a binary format. In Query/View mode, provide a button to preview the image inside a popup/modal.
* **ProductTypeId:** Filter the selectable ProductTypes to match ONLY the currency defined in the Order.
* **ListPrice:** Keep as-is.
* **ListPriceTaxWithTax (Rename from ListPriceTax):** This is an editable field. Automatically calculate it on the fly when typing the item based on the order's tax rate: `ListPriceTaxWithTax = ListPrice + TaxAmount`.
* **ServiceFeeInLocal:** Default value is loaded automatically based on the selected `ProductType`. If `ProductType` changes, refresh this value. *Must be included in the Query view.*
* **Total:** Visual/calculated field only (Include in the model as an unmapped/computed property, do not persist in DB). Formula: `(ListPriceTaxWithTax converted using ExchangeRate) + ServiceFeeInLocal`. *Must be included in the Query view.*
* **AgreedPriceInLocal:** - Default value matches `Total`.
    - It is manually editable.
    - If `ListPrice` changes, update this field.
    - If `ServiceFeeInLocal` changes, update this field.
    - If the Order's `ExchangeRate` changes, recalculate this field for ALL items in the order.
    - *Must be included in the Query view.*
* **RealPrice:** Default value matches `ListPrice`. It is manually editable. *Must be included in the Query view.*
* **EstimateShipping:** Default value is loaded automatically based on `ProductType`. It is manually editable. *Must be included in the Query view.*

### 5. Order Calculation Suite (Formulas)
Whenever an order-level recalculation is triggered, update the fields sequentially using these formulas:

1.  **TotalAgreedPriceInLocal (NEW FIELD - Add to Order Model and DB Schema):** Sum of all items' `AgreedPriceInLocal`.
2.  **ShippingAmountToCR:** Sum of all items' `EstimateShipping`.
3.  **TotalWithoutTaxes:** Sum of all items' `RealPrice`.
4.  **TaxesAmount:** `(TotalWithoutTaxes - DiscountAmount) * TaxPercentage`.
5.  **TotalToPayToSupplier:** `ShippingAmountIntern + TotalWithoutTaxes + TaxesAmount - DiscountAmount`.
6.  **TotalOfTheOrder:** `TotalToPayToSupplier + ShippingAmountToCR`.
7.  **EstimatedProfitInLocal:** `TotalAgreedPriceInLocal - (TotalOfTheOrder * ExchangeRate)`.
